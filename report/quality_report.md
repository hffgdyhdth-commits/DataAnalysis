# Отчёт о качестве данных

## 1. Введение

**Датасет:** Metacritic Games Dataset  
**Источник:** [Metacritic](https://www.metacritic.com/)  
**Размер:** 13 913 строк, 16 столбцов  

**Предметная область:** Набор данных содержит информацию о видеоиграх с сайта Metacritic, включая рейтинги критиков (metascore) и пользователей (userscore), даты выпуска, платформы, жанры, разработчиков и издателей. Данные охватывают игры с 1998 по 2025 год.

**Условный заказчик:** Игровые издательства, аналитические компании в игровой индустрии, маркетинговые агентства.

**Возможные задачи ИАД:**

- **Анализ факторов успеха игр:** Исследование зависимости рейтингов от жанра, платформы, разработчика, года выпуска и других характеристик для выявления ключевых факторов коммерческого и критического успеха.
- **Сегментация рынка и прогнозирование:** Кластеризация игр по характеристикам и рейтингам для выявления рыночных сегментов. Построение моделей прогнозирования пользовательских рейтингов на основе мета-оценок и других признаков.
- **Анализ репутации разработчиков и издателей:** Оценка качества продукции различных студий и издателей на основе средних рейтингов их игр, выявление лидеров и аутсайдеров в разных жанрах.

## 2. Паспорт датасета

| Столбец | Тип | Описание | Примечания |
|---------|-----|----------|------------|
| id | int | Уникальный идентификатор игры | Числовой ID |
| title | string | Название игры | Может содержать год выпуска в скобках |
| releaseDate | string | Дата выпуска | Формат YYYY-MM-DD |
| rating | string | Возрастной рейтинг ESRB | M, T, E, E10+, AO, RP, K-A |
| genres | string | Жанр(ы) игры | Может содержать несколько жанров |
| description | string | Описание игры | Может содержать технические детали |
| platforms | string | Платформы выпуска | Перечисление через запятую |
| metascore | float | Рейтинг критиков (0-100) | Пропущено 1 значение |
| metascore_count | int | Количество оценок критиков | Целое число |
| metascore_sentiment | string | Категория рейтинга критиков | Universal acclaim, Generally favorable и др. |
| userscore | float | Рейтинг пользователей (0-100) | Пропущено 0 значений |
| userscore_count | int | Количество оценок пользователей | Целое число |
| userscore_sentiment | string | Категория рейтинга пользователей | Universal acclaim, Generally favorable и др. |
| platform_metascores | string | Рейтинги по платформам | Перечисление через запятую |
| developer | string | Разработчик игры | Пропущено 9 значений |
| publisher | string | Издатель игры | Пропущено 2 значения |

## 3. Основные проблемы качества данных

| № | Тип проблемы | Описание | Потенциальный риск | Предлагаемое решение |
|---|-------------|----------|-------------------|---------------------|
| 1 | Пропуски в категориальных данных | Значительное количество пропусков в столбце `rating` (16.55%) и `userscore_sentiment` (10.59%) | Невозможность анализа по возрастным категориям для части игр, потеря информации о восприятии пользователями | Для `rating`: попытка восстановления из внешних источников или удаление строк. Для `userscore_sentiment`: пересчет на основе числового `userscore` |
| 2 | Пропуски в текстовых данных | Пропуски в `description` (42), `developer` (9), `publisher` (2) | Потеря контекстной информации, невозможность анализа по разработчикам/издателям | Для `developer` и `publisher`: заполнение значением "Unknown". Для `description`: оставить как есть, так как не критично для основного анализа |
| 3 | Сложная структура данных | Столбцы `platforms` и `platform_metascores` содержат составные значения через запятую | Затруднение анализа по отдельным платформам, невозможность прямого использования в моделях | Создание отдельных таблиц для платформ или one-hot encoding для основных платформ |
| 4 | Выбросы в распределениях | Большое количество выбросов в `metascore` (359) и `userscore` (1620) по методу IQR | Искажение статистических показателей, влияние на модели машинного обучения | Анализ выбросов в контексте - возможно, это действительно экстремальные, но корректные значения (шедевры или провалы) |
| 5 | Несоответствие типов данных | Столбец `releaseDate` хранится как строка, а не дата | Затруднение временного анализа, невозможность использования временных функций | Преобразование в datetime формат |
| 6 | Отсутствие нормализации | Информация о платформах, жанрах и разработчиках не нормализована | Возможны вариации написания одних и тех же значений (например, "PC" и "Windows PC") | Стандартизация значений через маппинг или регулярные выражения |

## 4. План улучшения качества

**Приоритет 1:** Обработка пропусков в ключевых столбцах (`rating`, `userscore_sentiment`) и преобразование `releaseDate` в datetime.

**Приоритет 2:** Нормализация составных столбцов (`platforms`, `genres`) для удобства анализа и создания производных признаков.

**Приоритет 3:** Анализ и обработка выбросов в рейтингах, проверка на корректность экстремальных значений.

**Приоритет 4:** Стандартизация названий разработчиков и издателей для обеспечения консистентности.

## 5. Краткое резюме

Датасет содержит ценные данные о видеоиграх, включая профессиональные и пользовательские рейтинги, что делает его подходящим для анализа факторов успеха в игровой индустрии. Однако данные требуют значительной предварительной обработки: восстановления пропущенных значений в ключевых категориальных признаках, нормализации составных полей и стандартизации текстовых данных.

Особое внимание следует уделить столбцу возрастных рейтингов, где пропущено более 16% значений, что может существенно ограничить возможности сегментации по этой важной характеристике. Также стоит отметить высокую корреляцию между мета-оценками и пользовательскими оценками, что может быть использовано для построения моделей прогнозирования.

После выполнения этапа очистки и нормализации данные будут готовы для проведения глубокого анализа рынка видеоигр, выявления трендов и построения прогностических моделей.
